HaruFit 디자인 가이드라인

목적: 이 문서는 HaruFit 웹 애플리케이션의 일관된 디자인 시스템을 정의합니다. 모든 개발 및 디자인 작업은 이 가이드라인을 준수해야 합니다.

1. 브랜드 아이덴티티

1.1 브랜드 컨셉

HaruFit은 사용자가 자신의 운동 루틴을 체계적으로 관리하고 목표를 달성할 수 있도록 돕는 개인 맞춤형 피트니스 관리 플랫폼입니다.

1.2 디자인 철학

  • 명확성: 정보를 명확하고 직관적으로 전달
  • 일관성: 모든 화면에서 일관된 디자인 언어 사용
  • 접근성: 모든 사용자가 쉽게 접근할 수 있는 인터페이스
  • 모던함: 현대적이고 세련된 디자인
  • 기능성: 아름다움과 실용성의 균형

1.3 로고

HaruFit

로고는 항상 아이콘과 텍스트를 함께 사용하며, 주 색상(#6366f1)을 사용합니다.

2.4 컬러 사용 가이드

  • 텍스트는 최소 4.5:1 대비율을 유지해야 합니다.
  • 주 색상은 최대 3개까지만 사용합니다.
  • 배경은 주로 Gray 50 또는 White를 사용합니다.
  • 테두리는 Gray 200을 기본으로 사용합니다.

8. 반응형 디자인

8.1 브레이크포인트

디바이스 너비 설명
모바일 < 768px 스마트폰 세로 모드
태블릿 768px - 992px 태블릿 및 스마트폰 가로 모드
데스크톱 992px - 1200px 일반 데스크톱
대형 화면 > 1200px 대형 모니터

8.2 반응형 규칙

  • 모바일 (< 768px): 1컬럼 레이아웃, 아이콘 중심 네비게이션, 폰트 크기 축소
  • 태블릿 (768px - 992px): 2컬럼 레이아웃, 아이콘+텍스트 네비게이션
  • 데스크톱 (992px - 1200px): 3컬럼 레이아웃, 전체 네비게이션 표시
  • 대형 화면 (> 1200px): 4컬럼 레이아웃, 최대 너비 제한
  • 네비게이션은 모바일에서 아이콘만 표시하고 텍스트는 숨김
  • 폰트 크기는 모바일에서 약간 축소 (h1: 1.5rem, h2: 1.25rem)
  • 카드와 버튼의 패딩을 모바일에서 축소 (1.5rem → 1rem)
  • 통계 카드의 값 크기를 모바일에서 축소 (2.5rem → 2rem)

8.3 모바일 최적화 예시

/* 모바일 반응형 (768px 이하) */ @media (max-width: 768px) { .container { padding: 1rem; } .grid { grid-template-columns: 1fr; } h1 { font-size: 1.5rem; } h2 { font-size: 1.25rem; } .nav-item span { display: none; /* 모바일에서 텍스트 숨김 */ } .card { padding: 1rem; } } /* 작은 모바일 (576px 이하) */ @media (max-width: 576px) { h1 { font-size: 1.25rem; } .stat-value { font-size: 1.75rem; } .nav-menu { flex-direction: column; } }

8.4 Bootstrap 그리드 시스템 활용

Bootstrap 5의 반응형 클래스를 활용하여 모바일 최적화를 구현합니다:

  • col-12: 모바일에서 전체 너비
  • col-md-6: 태블릿 이상에서 50% 너비
  • col-lg-4: 데스크톱에서 33% 너비
  • d-none d-md-inline: 모바일에서 숨김, 태블릿 이상에서 표시
  • g-3: 그리드 간격 1rem

9. 접근성

9.1 색상 대비

  • 일반 텍스트: 최소 4.5:1 대비율
  • 큰 텍스트 (18px 이상): 최소 3:1 대비율
  • 인터랙티브 요소: 최소 3:1 대비율

9.2 키보드 네비게이션

  • 모든 인터랙티브 요소는 키보드로 접근 가능해야 합니다.
  • 포커스 표시가 명확해야 합니다.
  • Tab 순서가 논리적이어야 합니다.

9.3 스크린 리더

  • 모든 이미지에 alt 텍스트 제공
  • 폼 요소에 적절한 label 연결
  • 의미론적 HTML 태그 사용 (header, nav, main, footer 등)

9.4 포커스 스타일

:focus { outline: 2px solid var(--primary-color); outline-offset: 2px; } /* 또는 */ :focus-visible { box-shadow: 0 0 0 3px rgba(99, 102, 241, 0.3); }

10. 애니메이션

10.1 애니메이션 원칙

  • 빠르고 자연스러움: 0.2s - 0.3s 사이의 짧은 애니메이션
  • 의미 있는 움직임: 사용자에게 피드백을 제공하는 애니메이션
  • 성능 고려: transform과 opacity 사용 (GPU 가속)

10.2 일반적인 애니메이션

  • 호버 효과: transform: translateY(-2px) + 그림자 증가
  • 로딩: 페이드 인/아웃 또는 스피너
  • 페이지 전환: 페이드 효과 (0.3s)
  • 알림: 슬라이드 다운 + 페이드 인
/* 호버 애니메이션 */ .card { transition: transform 0.2s, box-shadow 0.2s; } .card:hover { transform: translateY(-2px); box-shadow: 0 4px 8px rgba(0,0,0,0.1); }

11. 모범 사례

11.1 DO (해야 할 것)

  • ✓ 일관된 간격과 정렬 사용
  • ✓ 명확한 계층 구조 유지
  • ✓ 충분한 여백 확보
  • ✓ 적절한 색상 대비 유지
  • ✓ 반응형 디자인 고려
  • ✓ 접근성 가이드라인 준수

11.2 DON'T (하지 말아야 할 것)

  • ✗ 너무 많은 색상 사용
  • ✗ 일관성 없는 간격
  • ✗ 너무 작은 폰트 크기
  • ✗ 과도한 애니메이션
  • ✗ 접근성을 무시한 디자인
  • ✗ 모바일을 고려하지 않은 레이아웃
